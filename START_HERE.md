🚀 THREE WAYS TO GET STARTED

OPTION 1: LOCAL (Fastest for development)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Download the gagcoffee/ folder
2. cd gagcoffee
3. bash setup.sh
4. Get MongoDB URI from mongodb.com/atlas
5. Edit backend/.env (paste MongoDB URI)
6. Terminal 1: cd backend && python manage.py runserver
7. Terminal 2: cd frontend && python -m http.server 5500
8. Open http://localhost:5500

Time: 5 minutes ⏱️

OPTION 2: DOCKER (Best for consistency)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Install Docker Desktop
2. cd gagcoffee
3. docker-compose up
4. Open http://localhost:5500

Time: 3 minutes ⏱️

OPTION 3: VERCEL (Go live immediately)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Push gagcoffee/ to GitHub
2. Go to vercel.com
3. Import the GitHub repo
4. Add environment variables
5. Click Deploy

Time: 15 minutes ⏱️
See DEPLOYMENT.md for detailed steps

🔐 SECURITY CHECKLIST

Before Production:
  ☐ Change SECRET_KEY to random string
  ☐ Set DEBUG = False
  ☐ Setup MongoDB Atlas with strong password
  ☐ Whitelist IPs in MongoDB (not 0.0.0.0)
  ☐ Create Django superuser
  ☐ Update ALLOWED_HOSTS with your domain
  ☐ Add frontend URL to CORS_ALLOWED_ORIGINS
  ☐ Enable HTTPS (automatic on Vercel)
  ☐ Setup automated backups
  ☐ Monitor error logs
